# cdesign project

Generated by cdesign runtime. Follow these constraints when modifying.

## Rules
- One spectacle per viewport (no stacking shader + particles + magnetic in same section)
- Transform + opacity only for animations
- One scroll clock: Lenis → GSAP ticker (autoRaf: false)
- Stagger 0.015–0.025 (target 0.02)
- Default ease: cubic-bezier(0.16, 1, 0.3, 1)
- Use existing components from components/ — do not invent new animation architecture
- PerformanceMonitor required on any R3F Canvas, DPR capped (default 1.5, adaptive 1–2)
- Use VisibilityGate around heavy offscreen sections
- Motion hierarchy: Tier 1 hero > Tier 2 transitions > Tier 3 hover > Tier 4 ambient
- Magnetic / tilt effects gated by `(pointer: fine)` — coarse pointers get tap-only
- Tap targets >= 44px on real CTAs; focus-visible ring on every interactive control
- Lenis `syncTouch: false` (iOS scroll inertia must be native)

## Pre-handoff gate (run all five, all must pass)
```bash
npm run lint          # warnings allowed, errors not
npm run typecheck     # tsc --noEmit
npm run build         # next build
npm run audit:cdesign # static blocker scan (h-screen, syncTouch, Canvas w/o PerformanceMonitor, key={index}, ...)
npm run pack:cdesign  # refreshes cdesign-starter.zip in the project root
```
After `npm run build`, re-run `npm run lint` to catch anything regenerated. If browser screenshot QA is available, sweep the viewports listed in `README.md` (Visual QA) — never ship without checking mobile (375/390/430).

When you downgrade behavior for mobile or coarse pointers, do NOT change visual identity. Keep the metaphor, type system, color, and section rhythm. Only reduce motion intensity, DPR, or shader complexity.

## Do NOT
- Rewrite project from scratch on edits
- Use framer-motion package (use motion/react)
- Use h-screen (use min-h-[100dvh])
- Use transition: all (list specific properties)
- Use raw 100vh (use 100dvh)
- Add key={index}
- Animate layout properties (width/height/top/left/margin/box-shadow/filter)
- Use useState for pointer tracking (use MotionValue)
- Stack backdrop-filter on every card in a grid (kills mobile FPS — gate by device tier)
- Leave permanent will-change on reusable components (use `useTemporaryWillChange`)
